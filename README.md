## 📝 文档规范

### 命名规则
- `项目名_文档类型_版本号.扩展名`
- 示例：`光与朽_技能系统设计案_V3.0.docx`

### 版本管理
- **最新版**：放在对应功能文件夹
- **旧版本**：放入"旧版本归档"文件夹
- **迭代记录**：每次大版本更新需在归档中注明变更点

### AI对话记录
- 按角色分类存放（策划/程序/美术/模型）
- 不合并，保留原始记录
- 方便追溯思考过程

### 创意库使用
- 每日游戏拆解自动追加到`每日游戏拆解记录.md`
- 新创意随时补充到创意库
- 热门游戏分析报告归入创意库参考

## 🎯 使用流程

### 每日工作
1. **行业调研** → 存入 `03_市场分析/`
2. **游戏拆解** → 追加到 `02_创意库/每日游戏拆解记录.md`
3. **项目开发** → 更新对应项目文件夹

### 版本迭代
1. 新版本的文档放入对应功能文件夹
2. 旧版本移入 `08_旧版本归档/`
3. 更新 `旧版本相关文档汇总.md`
